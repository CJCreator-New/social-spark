**Root cause:** The regenerate handler references `longFormPlatform` which is either out of scope or never declared in the component where it's called.

```tsx
const handleRegenerate = async (postId: string) => {
  const platform = selectedPlatform; // ← read from component state
  ...

```

```tsx
try {

}
```

### BUG-03 · Raw `**markdown**` Appears in Post Body Text

**Symptom:** Posts show literal `**bold**` asterisks in the rendered body. Affects 5 of 7 posts, especially list-format posts.

**Two-part fix:**

**Part A — Fix the AI prompt** to explicitly forbid markdown syntax:

```ts
// In your generation prompt (wherever you build the system/user message):
const systemPrompt = `
You are a LinkedIn content writer. 
CRITICAL FORMATTING RULES:
- Do NOT use any markdown syntax: no **bold**, no *italic*, no ## headers
- For bullet points, use only: • or → as plain text characters
- Write in plain text only. LinkedIn renders plain text — markdown symbols appear literally.
- Example WRONG: "**Key insight:** This matters"
- Example CORRECT: "Key insight: This matters"
`;
```

**Part B — Strip any leaked markdown as a safety net** in your post rendering:

```tsx
// src/lib/formatPost.ts (create if it doesn't exist)
export function stripMarkdown(text: string): string {
  return text
    .replace(/\*\*(.*?)\*\*/g, '$1')   // **bold** → bold
    .replace(/\*(.*?)\*/g, '$1')         // *italic* → italic
    .replace(/#{1,6}\s/g, '')            // ## headers → remove #
    .replace(/`(.*?)`/g, '$1');          // `code` → code
}

// Use in your post display component:
<p>{stripMarkdown(post.body)}</p>
```

### BUG-04 · "Copy for LinkedIn" Copies Raw Markdown by Default

**Symptom:** The primary copy button pastes `**text**` into LinkedIn. The plain text option is buried in a dropdown.

**Fix:** Apply `stripMarkdown()` (from BUG-03) to ALL copy paths, including the primary button.

```ts
// src/lib/platformCopy.ts
import { stripMarkdown } from './formatPost';

export function copyForPlatform(post: Post, platform: string): string {
  const cleanBody = stripMarkdown(post.body);
  const cleanHook = stripMarkdown(post.hook);

  return [cleanHook, cleanBody, post.cta, post.hashtags.join(' ')].join('\n\n');
}
```

**Also:** Make "Copy as plain text" the default visible button. Move the formatted version into the dropdown instead — most users never need raw markdown.

### BUG-14 · British Spelling Used in India-Facing Product

**Fix:** Add to your AI generation system prompt:

```
Use Indian-English spelling conventions (same as American English).
Examples: "optimizing" not "optimising", "organizing" not "organising", "recognizing" not "recognising".
```

**Also add a post-processing pass:**
```ts
const britishToIndian: Record<string, string> = {
  'optimising': 'optimizing',
  'organising': 'organizing',
  'recognising': 'recognizing',
  'utilising': 'utilizing',
  'analysing': 'analyzing',
  'emphasising': 'emphasizing',
};

export function fixSpelling(text: string): string {
  let result = text;
  for (const [brit, indian] of Object.entries(britishToIndian)) {
    result = result.replace(new RegExp(brit, 'gi'), (match) =>
      match[0] === match[0].toUpperCase() ? indian.charAt(0).toUpperCase() + indian.slice(1) : indian
    );
  }
  return result;
}
```
